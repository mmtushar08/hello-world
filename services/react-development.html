FAQ

Common questions

Do you use the Next.js App Router or Pages Router? +
All new projects use the App Router (Next.js 13+). It's the direction Vercel and the React team are building toward, and Server Components are now stable and production-ready. We do support Pages Router maintenance for existing projects.
What's the minimum engagement size for React development? +
Our minimum project engagement is $25,000. This ensures we can do proper discovery, architecture, and deliver production-quality work with documentation. Smaller scopes are often better served by a retainer model for an existing project.
How do you handle state management? +
We default to React Server Components + React Query for server state, and Zustand for UI state. We avoid Redux unless the codebase already uses it and migration isn't in scope. We don't use Context for high-frequency state updates.
Can you build both the frontend and backend API? +
Yes. We build full-stack Next.js applications with API routes, server actions, and database layers (Prisma + PostgreSQL or Supabase). We can also build standalone Node.js APIs that serve your React frontend.
What does a Next.js project cost? +
A focused Next.js marketing site or landing page system starts at $25,000. A full SPA or dashboard application with auth, APIs, and database starts at $50,000. Enterprise-scale applications with multiple integrations, role-based access, and complex data pipelines start at $100,000+.
